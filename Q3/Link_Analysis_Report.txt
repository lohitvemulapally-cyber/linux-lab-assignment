=== Question 3: File System and Link Analysis ===

1. Inode Numbers of All Files
Command: ls -li
Output (before deletion):
44894 -rw-r--r-- 2 lohit lohit 45 Jul  3 08:11 hardlink.txt
44894 -rw-r--r-- 2 lohit lohit 45 Jul  3 08:11 original.txt
44932 lrwxrwxrwx 1 lohit lohit 12 Jul  3 08:11 symlink.txt -> original.txt
Observation: original.txt and hardlink.txt share the identical inode number 44894, proving they are two names for the exact same data on disk. symlink.txt has its own separate inode (44932) because it is a distinct file that merely stores a path/reference to original.txt.

2. Comparison Between Hard Link and Symbolic Link
Command: stat original.txt; stat hardlink.txt; stat symlink.txt
Output:
original.txt & hardlink.txt: Inode 44894, Size 45 bytes, Type "regular file", Links: 2, Permissions 0644
symlink.txt: Inode 44932, Size 12 bytes, Type "symbolic link", Links: 1, Permissions 0777 (lrwxrwxrwx)
Observation: A hard link shares the same inode, size, and metadata as the original file and increases the inode's link count. A symbolic link is a completely separate small file that only stores the path string of its target ("original.txt", 12 bytes), has its own inode, and always displays full permissions (0777) since actual access control is determined by the target file, not the symlink itself.

3. Effect of Deleting the Original File
Command: rm original.txt; cat hardlink.txt; cat symlink.txt; ls -li
Output:
This is the original file for link analysis.
Extra line added via original.
cat: symlink.txt: No such file or directory

44894 -rw-r--r-- 1 lohit lohit 76 Jul  3 08:14 hardlink.txt
44932 lrwxrwxrwx 1 lohit lohit 12 Jul  3 08:11 symlink.txt -> original.txt
Observation: After deleting original.txt, hardlink.txt still contains and displays the full file content because the underlying inode (44894) still has one remaining link (the hardlink itself) keeping the data alive on disk - the link count dropped from 2 to 1 instead of the data being deleted. In contrast, symlink.txt became a "dangling" or broken symlink because it never held the actual data, only a path reference to a filename that no longer exists.
